BabyStepsMilestone Tracker App — Wireframe SpecificationIndian Academy of Paediatrics (IAP) Standards · v1.0Senior UX Designer — April 20261.  Document purposeThis wireframe specification defines the information architecture, screen flows, interaction patterns and UX decisions for BabySteps — a mobile app that helps Indian parents track their child&apos;s developmental milestones against IAP and Indian normative standards. The document is intended for product, engineering and clinical review teams.2.  User flow overviewThe app follows a linear onboarding-to-insights funnel for first-time users, which collapses into a persistent timeline home on all subsequent visits:Onboarding — baby profile creationMilestone quiz — age-contextualised questionsAssessment — scored results with IAP comparisonTimeline home — vertical tendril visualisation (persistent home)On second visit onwards, users land directly on the Timeline (Screen 4). The quiz can be re-triggered from the profile screen or via a smart prompt when the app detects the baby has entered a new age window.3.  Screen 1 — Onboarding3.1  PurposeCollect the minimum required data to personalise milestone calculations. The screen must feel warm and non-clinical — parents should feel celebrated, not interrogated.3.2  UI componentsHero cardGradient card (IAP green ramp) with app mascot illustration and welcoming copy. Localized to Hindi/English toggle.Baby&apos;s nameFree text input. Used for personalisation throughout the app.Date of birthDate picker. Triggers corrected-age calculation flag if premature toggle is on. Drives all milestone window computations.Birth weightNumeric input in kg. Used for weight-for-age growth chart overlay (future feature).GenderSegmented control: Boy / Girl / Other. IAP norms differ slightly by gender; this informs percentile calculations.Premature toggleYes/No binary toggle. When Yes, a secondary field requests gestational age in weeks. All milestone windows shift to corrected age per IAP preterm guidelines.Progress dots3-dot indicator for multi-step onboarding (Profile → Confirm → Done).3.3  Key UX decisionsPremature correction: If gestational age &lt; 37 weeks, the app silently applies corrected age to all milestone windows. A persistent pill badge on the timeline reads Corrected age to remind the parent.No registration wall: Parents can complete onboarding and see their first assessment without creating an account. Account creation is prompted only when they want to save data to cloud.Tone: Copy uses the baby&apos;s name from the first field onward (&quot;Let&apos;s set up Aarav&apos;s profile&quot;) to create warmth from step one.4.  Screen 2 — Milestone quiz4.1  PurposeCollect the parent&apos;s observation of which milestones the child has or has not achieved. The quiz must feel playful, not clinical. Questions are pre-filtered to the child&apos;s current age window so every question feels personally relevant.4.2  Question logicQuestion setDynamically generated from the IAP milestone database filtered to the baby&apos;s corrected age ±2 months. Typically 7–10 questions per session.Domains coveredGross motor, fine motor, language/communication, social/emotional, cognitive. Each question is tagged with its domain and shown as a subtle label below the question text.IAP window displayEach question shows the expected IAP window (e.g., &quot;Expected 4–6 months&quot;) so parents understand the clinical benchmark.Answer options4-option radio list: (1) Yes, confidently; (2) Sometimes / partially; (3) Not yet; (4) Not sure. Partial credit scoring is applied for option 2.ProgressLinear progress bar at the top showing n of total questions. Baby name + age tag visible throughout for context.NavigationForward-only during first quiz. Back navigation allowed if parent taps a previous answered dot.4.3  Key UX decisionsNo skip allowed: All questions must be answered to generate a complete assessment. &quot;Not sure&quot; is always available as a valid option.Gamified pacing: Micro-animation on each answer selection (radio fills, subtle haptic). No long loading states — transitions are instant.Contextual images: Optional small illustration next to each question showing the milestone action (e.g., baby rolling over). Images use IAP-approved simplified drawings, not photorealistic stock.5.  Screen 3 — Assessment results5.1  PurposeTranslate quiz answers into an actionable assessment: show parents where their child stands relative to IAP norms, surface science-backed suggestions for any lags, and provide a clear paediatrician prompt for serious concerns.5.2  UI componentsSummary cardScore out of total (e.g., 5/7 on track) with a colour-coded headline. Background gradient shifts: all-green for no lags, amber for watches, red-tinted for any lagging milestone.Milestone rowsOne card per milestone assessed. Each card shows: milestone name, domain tag, IAP expected window, parent&apos;s entry (achieved at X months OR &quot;not yet&quot;), and a status badge.Status badges3-tier system: On track (IAP green) = achieved within window; Watch (amber) = approaching deadline, not yet achieved; Lagging (coral/red) = past IAP window, not yet achieved.Suggestion boxesCollapsible panel inside Watch and Lagging rows. Contains 2–3 actionable suggestions sourced from IAP and WHO guidelines. Language is plain, parent-friendly, non-alarmist.Paediatrician CTASurfaces only on Lagging milestones. Tapping opens a modal with a preformatted WhatsApp/SMS message the parent can send to their doctor, or a list of IAP-certified paediatricians nearby (location permission).CTA to timelinePrimary button at bottom navigates to Screen 4 (the timeline home).5.3  Key UX decisionsNon-alarmist copy: &quot;Watch&quot; milestones use encouraging language (&quot;Not there yet — here&apos;s what to try&quot;). &quot;Lagging&quot; milestones are honest but paired immediately with an action (&quot;We recommend a check-up&quot;).IAP citation visible: Each suggestion includes a small footnote citing the IAP guideline chapter so parents can verify. This builds trust, especially for first-generation smartphone users.Comparison column: Parent&apos;s entry and IAP window shown side by side on every row for clear visual comparison without requiring the parent to do mental arithmetic.6.  Screen 4 — Timeline home6.1  PurposeServe as the persistent home screen after first-run quiz completion. Provides an at-a-glance developmental story from birth to the present, and a forward-looking view of upcoming milestones. The tendril-grow scroll animation makes the experience feel alive and celebratory.6.2  Timeline anatomySpineA 2px vertical line running the full height of the screen, anchored to the left. Represents the passage of time.NodesCircular dots attached to the spine. Filled teal = achieved. Outlined teal with glow ring = most recent hit. Outlined amber = next upcoming. Outlined grey dashed = future milestones.Milestone cardsAttached to each node via a 16px horizontal connector. Cards show: milestone name, domain tag, achieved date or expected window.Month labelsAge section headers (&quot;3 months&quot;, &quot;6 months&quot;, &quot;9 months&quot;) float left of the spine as subtle grey labels. These act as scroll anchors.Most recent hitThe most recently achieved milestone is highlighted with a teal card background and a 🎉 label. This is the landing point after quiz completion — the screen scrolls to this card on entry.Up next cardThe single next expected milestone shown in amber. One card only — subsequent upcoming milestones are shown as future (tendril) nodes.Future milestonesRendered with dashed card borders and decreasing opacity (70%, 55%, 40%) as they recede into the future. On scroll, they bloom in with the tendril animation.6.3  Tendril scroll animationThe tendril effect is the signature interaction of BabySteps. It is implemented using SVG stroke-dashoffset animation triggered by IntersectionObserver.Mechanism: Each section of the spine between two nodes is rendered as an individual SVG path with a decorative vine/tendril overlay. The path starts at stroke-dashoffset: 100% (invisible) and animates to 0% (fully drawn) when it enters the viewport.Vine aesthetic: The tendril is a thin organic curved path (S-curve or sinuous wave) drawn alongside the spine, in a lighter green. Small leaf shapes (simple ellipse pairs) appear at each node as the tendril reaches it.Performance: Animation uses CSS transform and stroke-dashoffset only (compositor-thread safe). No JavaScript animation loops. IntersectionObserver threshold: 0.2.Reduced motion: The animation is wrapped in @media (prefers-reduced-motion: no-preference). Users with reduced motion see cards fade in instead.6.4  Navigation &amp; interactionsBottom nav3 tabs: Timeline (home), Assessment (last results), Profile (baby info + add siblings). Active tab uses teal fill icon.Tap a milestone cardOpens a detail sheet showing: full milestone description, IAP source, parent&apos;s log entry, and a note field for the parent to add a memory or photo.Tap the &apos;Up next&apos; cardOpens the suggestion panel for that milestone — same content as the Watch panel in the assessment screen.Add milestone manuallyFAB (+) in the bottom-right corner. Allows marking a milestone as achieved outside the quiz cycle. Triggers a confetti micro-animation.Scroll to top shortcutTapping the baby avatar in the header scrolls smoothly to birth (top of timeline).Share milestoneLong-press on any achieved milestone card offers a share sheet to generate a keepsake card (name, milestone, date) for WhatsApp.7. IAP milestone data modelThe following domains and milestone categories must be covered in the quiz and timeline. The app must support milestones from birth to 5 years (60 months).Gross motorHead control, rolling, sitting, crawling, pulling to stand, walking, running, jumping, climbing stairsFine motorPalmar grasp, pincer grasp, banging objects, stacking blocks, drawing lines/circles, using scissors, writingLanguage / communicationSocial smile, cooing, babbling, first words, two-word phrases, sentences, following commands, naming objectsSocial / emotionalEye contact, social smile, stranger anxiety, separation anxiety, parallel play, cooperative play, empathyCognitiveObject permanence, cause-effect, pretend play, colour/shape sorting, counting, problem solvingAge rangeBirth to 60 months (5 years). Quiz sessions auto-triggered at 2, 4, 6, 9, 12, 18, 24, 36, 48, 60 months.Data sourceIAP Growth Monitoring and Developmental Assessment Guidelines (2022 update). Cross-referenced with WHO Motor Development milestones.Preterm adjustmentAll milestone windows shifted by (40 - gestational age at birth) weeks for babies born before 37 weeks.9.  Edge cases &amp; error statesNo internet on first launch: Onboarding and quiz complete offline. Assessment runs on-device. Data syncs to cloud when connection is restored.Multiple children: Profile screen supports up to 5 children. Timeline tab switches between children via avatar row at the top.Very premature babies (&lt; 28 weeks): A dedicated copy variant acknowledges the NICU journey and adds a note that corrected age may be applied for up to 24 months. Paediatrician guidance is surfaced more proactively.Parent marks milestone achieved retroactively: Log entry allows backdating. If backdated entry falls within IAP window, status shown as on track. If after window, marked as &quot;Achieved with delay&quot; (not lagging, since it is now achieved).All milestones on track: Assessment screen shows a celebration state: confetti, full-green score card, encouraging message, and a share card option.Language support: v1 targets English and Hindi. All milestone text and suggestion content must be available in both languages. Language auto-detected from device locale, switchable in settings.10.
